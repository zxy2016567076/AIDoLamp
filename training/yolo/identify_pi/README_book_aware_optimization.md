# 树莓派YOLOv8优化说明

## 主要优化内容

此版本 `main_optimized_book_aware.py` 针对树莓派性能和"book"类的误识别问题进行了全面优化：

### 1. 性能优化提高帧率

- **减小输入尺寸**: 默认使用320×320而非640×640，显著提高推理速度
- **ONNX运行时优化**: 
  - 启用所有图优化
  - 设置4线程处理（适合树莓派4核心）
  - 使用顺序执行模式减少内存占用
- **相机参数优化**:
  - 优化曝光时间(10000)，减少运动模糊
  - 帧率控制，平衡性能与稳定性
- **处理流程优化**:
  - 轻量级预处理与后处理流程
  - 首次过滤提前减少处理数据量

### 2. "Book"类误识别解决方案

- **类别特定阈值**:
  - 为"book"类设置更高的默认置信度阈值(0.45)，减少误识别 
  - 其他类别使用较低阈值(0.25)，保持正常检测灵敏度
- **几何特征过滤**:
  - 基于长宽比过滤(书本通常在0.5-2.0范围内)
  - 基于面积比例过滤(忽略小于图像1%面积的检测，减少噪声)
- **运行时阈值调整**:
  - 使用键盘'+'和'-'实时调整Book类阈值，方便现场调试

### 3. 增强的显示信息

- **性能指标**:
  - FPS实时显示与平滑处理
  - 分解预处理、推理、后处理时间(ms)
  - 总处理时间统计，帧计数
- **检测分析信息**:
  - "book"类显示长宽比(AR)和面积比(Area)
  - 当前阈值设置显示
  - 时间戳显示
- **UI优化**:
  - 半透明性能信息背景提高可读性
  - 为不同类别使用不同颜色
  - 优化标签放置，提高可见度

## 使用说明

### 运行程序

```bash
python main_optimized_book_aware.py
```

### 键盘控制

- **q键**: 退出程序
- **+键**: 增加book类阈值(减少误识别，也可能减少正确检测)
- **-键**: 降低book类阈值(增加检测敏感度，也可能增加误识别)

### 模型支持

脚本首先尝试加载优化的ONNX模型:
```
runs/detect/my_custom_model/weights/best_quant.onnx
```

如找不到ONNX模型，会尝试使用PyTorch模型(`best.pt`)，但会提示转换建议。

## 优化建议

如果在树莓派上仍然遇到性能问题:

1. **进一步减小输入尺寸**: 
   - 可将`input_size`调整至(256, 256)或更小
   - 在`OptimizedDetector`初始化时修改参数

2. **进一步调整阈值**:
   - 如果误识别问题严重，增加book类阈值(0.5-0.6)
   - 如果漏检，考虑降低阈值但启用几何过滤

3. **训练集改进**:
   - 添加更多"非book但容易误识别为book"的负样本
   - 增加不同角度、光照条件下的book样本
   - 考虑对数据集进行类别平衡(确保book类样本与其他类比例合理)

4. **运行时优化**:
   - 关闭树莓派上其他非必要进程
   - 考虑超频树莓派(需要散热方案)
   - 优化相机设置，减少捕获延迟

## 调试提示

- 使用显示的时间分解来确定瓶颈(预处理/推理/后处理)
- 观察book类的AR和Area指标，帮助优化几何过滤参数
- 如果特定场景下误识别率高，记录这些场景的图像，添加到训练集中
